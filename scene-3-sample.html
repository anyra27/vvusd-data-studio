How the Gem works · every time

Three phases. No skipping.

Phase 01 · Acknowledge & Clean

Spot the mess. Propose a fix. Wait.

The Gem confirms what it sees, lists the data problems, and offers to clean them. It will not analyze before you say go.

Phase 02 · Interview

Three short questions before building.

Audience? Big question? Anyone or anything specific to watch? Default audience pulls from your TEACHER.md.

Phase 03 · Build

Complete HTML, strict UI Library.

A copyable HTML document using only .ui-* classes from your library, Chart.js with the brand gradient, an audience callout that ends with the move to make next.

Try it now

Open your Gem and run this.

  1. Open your Classroom Data Analyzer Gem.
  2. Drag sample-classroom-data.csv into the chat. (Or drag-drop from where you saved it in Scene 02.)
  3. Read what the Gem says back — it should call out the messes you saw above and propose a cleanup.
  4. Reply go. The Gem cleans, then asks the three Phase 2 questions.
  5. Answer: audience = PLC, big question = Where's the gap on Lab 03?, watch = Marcus Chen.
  6. The Gem builds a complete HTML dashboard. Save it as sample-dashboard.html and open it in a browser. Make sure ui-library.css is in the same folder — it's only there if you saved it from Scene 02.
  7. Once you have the sample working, you're ready for the real thing. Move on.
