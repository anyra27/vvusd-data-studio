Five things, no zip required.

Each item below has its own action — copy or download. The label on the right tells you exactly where it goes in Gemini's Gem builder. Click each row to mark it done.

01
Gem Instructions → Gemini · Instructions field
The full system prompt that runs your Gem — three phases (clean → interview → build), strict UI Library, audience voices.
02
TEACHER.md → Gemini · Knowledge files
Your classroom context — grade band, subject, class size, default audience. Edit it before uploading. The Gem reads it before answering anything.
Download file
03
Sample Classroom CSV → Drop into the chat
Lab 03 data for 28 students — intentionally messy (mixed dates, name variants, "absent" as text, blanks). You'll use it in Scene 03 to test your Gem.
Download CSV
04
UI Library Reference → Open + leave open
Live demo of every .ui-* class your Gem will use. Keep it open while you build — what "good" looks like.
Open in new tab
05
ui-library.css → Save next to your dashboard HTML
Single bundled stylesheet. Never goes into Gemini. Save it next to the HTML your Gem produces so the styles render when you open the dashboard.
Download CSS
Once you have all five

Build your Gem in Gemini

01
Go to gemini.google.com, open Gems, click Create new Gem.
02
Name it Classroom Data Analyzer.
03
Paste the Gem Instructions (step 01 above) into the Instructions field.
04
Open TEACHER.md in a text editor — fill in your grade, subject, class size, audience.
05
Drag TEACHER.md into the Gem's Knowledge files section.
06
Hit Save. Test it: "Hi! Ready when you are." — your Gem should ask for your data.
